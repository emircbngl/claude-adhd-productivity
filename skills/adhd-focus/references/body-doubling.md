# Body doubling — presence, not supervision

Body doubling is the single most reliable ADHD focus tool: doing a task in the *presence* of another person dramatically lowers activation energy and sustains attention. The magic is **presence and gentle accountability**, not help or instruction. You can be that presence — synchronously in the conversation, or via timed check-ins.

The double does **not** need to understand the task, do the task, or watch every move. They just need to be *there* and to expect an update.

## The sprint protocol

1. **Agree on ONE step.** Not the task — the single next action. ("Just open the file and write the title.")
2. **Set a timer.** Say the length out loud. Start short: 10–15 minutes. (You can't literally run a timer — have the user set a phone/system timer, and you hold the frame.)
3. **Say "go."** Then get out of the way. Don't quiz, don't add tasks, don't chat. Silence is fine and good.
4. **Check in at the end.** "Time's up — how far did you get?" Celebrate whatever happened, even partial.
5. **Decide the next move together:** another sprint, a break, or a smaller step if they stalled.

## Timer guidance (defeats time blindness)

- Default first sprint: **15 minutes**. Easy to face.
- Adjust to the human, not the dogma: 10 if they're avoidant, 25 if they're warming up, 45+ if they hit flow (then *don't* interrupt — ride it).
- Always pair work with a **break**: a sprint without a stopping point invites burnout or avoidance. "15 on, 5 off."
- State time remaining when checking in: "~5 min left, keep going."

## Check-in language (warm, low-pressure)

- "Still on step 2? Want me to shrink it?"
- "How's it going — moving, stuck, or distracted?" (all three are fine answers)
- "What's the one thing in front of you right now?"
- After: "Nice — that's done. Quick break or straight into the next?"

## Async / over-time body doubling

If the user steps away to work, set an explicit return point and a check-in:
- "Go do the 15. Come back and tell me what got done — I'll be here."
- Leave a Resumption point in `FOCUS.md` so re-entry is frictionless.
- When they return, open with the celebration + the next single action.

## What NOT to do

- Don't supervise or interrogate. Pressure triggers shame and avoidance.
- Don't pile on tasks mid-sprint. New ideas → parking lot, not the current step.
- Don't judge a small result. Partial progress during a sprint is a *win*, not a shortfall.
- Don't keep them past flow's natural end, and don't cut off hyperfocus that's working.

## Quick script

> "Let's body-double this. One step: open the doc and write a junk first sentence — it's allowed to be terrible. Set a 15-min timer. Go — I'm right here. Tell me when the timer's up or if you get stuck."
